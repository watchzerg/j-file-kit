## 任务目标
对指定目录或代码文件进行 code review 和优化。

## Review检查项
1. **逻辑正确性**
   - 逻辑bug、边界条件、错误处理

2. **代码质量**
   - 命名规范、语义清晰（遵循项目规范）
   - 设计模式使用合理
   - 代码结构清晰、职责单一
   - 符合项目编码规范和架构原则
   - **注释清晰简练，描述设计意图**，为单元测试提供指导

3. **性能与可维护性**
   - 性能问题、可读性、可维护性
   - 重复代码或可抽象部分

## 优化范围
- 主要优化当前文件，必要时修改上下游依赖文件
- 大改动（涉及多文件或核心逻辑）需先说明方案和影响范围，等待确认

## 输出要求
- Review报告：问题列表（按优先级）、代码质量评估
- 优化执行：修复问题并说明主要优化点

## 重要原则
- 不要为了改而改：无问题直接告知，不强行修改
- 聚焦实际问题：只修复真正存在的问题，避免过度优化
