## 任务目标
生成代码变更摘要并提交git变更。

## 执行步骤
1. **分析变更**：使用 `git diff` 和 `git status` 分析所有代码变更
2. **生成摘要**：生成简洁清晰的commit message，包含：
   - 变更类型（新增/修改/删除/重构等）
   - 主要变更内容
   - 影响范围（如有）
3. **执行提交**：使用 `git commit` 提交变更

## 重要注意事项
- 项目已配置pre-commit hook，提交时可能会触发代码检查（如lint、类型检查等）
- 如果提交失败，需要：
  1. 查看错误信息，理解失败原因
  2. 修复所有检查失败的问题
  3. 重新执行提交
  4. 重复此过程直到提交成功
- commit message应遵循项目规范，简洁明了地描述变更内容
