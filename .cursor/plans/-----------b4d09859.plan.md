## 2. 核心设计模式

### 管道/过滤器架构（优化版）

```
Scanner → [Analyzer₁ → Analyzer₂ → Executor₁ → Executor₂ → ...] → Finalizers → Report
         └────────────── ProcessingContext 贯穿传递 ──────────────┘
```

- **Scanner**: 遍历文件目录，生成 `FileInfo` 流
- **Analyzer**: 分析文件，填充 `ProcessingContext`（如分类、番号提取）
- **Executor**: 根据 Context 执行操作（如重命名、移动）
- **Finalizer**: 全局后处理（如清理空目录）
- **Pipeline**: 协调流程，支持短路机制（Processor 可返回 `skip_remaining=True`）

### 数据模型（Pydantic v2）

- `FileInfo`: 文件基础信息（路径、扩展名）—— 轻量级，仅文件名信息
- `ProcessingContext`: 处理上下文（携带分析结果和中间状态）
                                                                                                                                - `file_info: FileInfo`
                                                                                                                                - `file_type: Literal["video", "image", "other"] | None`
                                                                                                                                - `serial_id: str | None` （提取的番号，统一大写）
                                                                                                                                - `target_path: Path | None` （计划的目标路径）
                                                                                                                                - `skip_remaining: bool` （短路标记）
- `ProcessorResult`: 单个处理器的处理结果（状态、消息、耗时）
- `TaskResult`: 单个文件的完整处理结果（包含所有 Processor 结果）
- `TaskReport`: 任务汇总报告（总数、成功、失败、统计）
- `TaskConfig`: 任务配置（从 YAML 加载）
                                                                                                                                - `global_config`: 全局配置（所有任务共享）
                                                                                                                                - `tasks`: 任务列表，每个任务包含类型和专属配置

### YAML 配置示例

`configs/task_config.yaml` 结构：

```yaml
# 全局配置
global:
  scan_root: /path/to/scan/directory
  log_dir: ./logs
  report_dir: ./reports
  
# 任务列表
tasks:
  # 任务1: 文件整理任务
  - name: video_file_organizer
    type: file_organize
    enabled: true
    config:
      todo_non_vidpic_dir: /path/to/todo-non-vidpic
      todo_vidpic_dir: /path/to/todo-vidpic
      video_extensions:
        - .mp4
        - .avi
        - .mkv
        - .mov
      image_extensions:
        - .jpg
        - .jpeg
        - .png
        - .webp
      serial_id_pattern: "[A-Za-z]{2,5}-\\d+"
  
  # 任务2: 数据库更新任务（未来扩展示例）
  - name: db_updater
    type: db_update
    enabled: false
    config:
      db_connection: postgresql://localhost/mydb
      update_fields:
        - file_path
        - file_size
        - last_modified
```

配置模型设计：

```python
class GlobalConfig(BaseModel):
    scan_root: Path
    log_dir: Path = Path("./logs")
    report_dir: Path = Path("./reports")

class FileOrganizeConfig(BaseModel):
    todo_non_vidpic_dir: Path
    todo_vidpic_dir: Path
    video_extensions: set[str]
    image_extensions: set[str]
    serial_id_pattern: str = r"[A-Za-z]{2,5}-\d+"

class TaskDefinition(BaseModel):
    name: str
    type: Literal["file_organize", "db_update"]
    enabled: bool = True
    config: dict[str, Any]  # 任务特定配置

class TaskConfig(BaseModel):
    global_: GlobalConfig = Field(alias="global")
    tasks: list[TaskDefinition]
```

## 4. 典型任务示例：文件整理

基于实际需求设计的完整任务流程：

### 任务目标

对指定目录进行视频/图片文件的番号提取和整理，非视频/图片文件移动到待处理目录。

### 处理流程

#### 4.1 分类处理

- **视频/图片文件**（通过扩展名过滤器识别）
                                                                                                                                - 进入番号提取流程
- **非视频/图片文件**
                                                                                                                                - 移动到 `todo-non-vidpic/` 目录
                                                                                                                                - 重名处理：追加后缀 `-Dup1234`（4位随机数）

#### 4.2 番号提取与重命名

- **番号格式**: `[A-Za-z]{2,5}-\d+`（例如：`ABCDE-0001`）
- **番号位置**: 可能在文件名的开头、中间或末尾
- **重命名规则**:
                                                                                                                                - 如果番号已在开头 → 跳过
                                                                                                                                - 否则 → 提取番号到开头，原位置替换为 `-serialId-`
                                                                                                                                - 重名处理：追加后缀 `-Dup1234`（4位随机数）

#### 4.3 后续分类

- **有番号的文件**: 保持在当前目录
- **无番号的文件**: 移动到 `todo-vidpic/` 目录
                                                                                                                                - 重名处理：追加后缀 `-Dup1234`（4位随机数）

#### 4.4 清理空目录

- 处理完成后，删除所有空目录

### 实现设计

此任务需要以下 Processors 协同工作：

1. `FileClassifier`: 判断文件类型（视频/图片/其他）
2. `SerialIdExtractor`: 提取番号并生成新文件名
3. `FileRenamer`: 执行重命名（带重名检测）
4. `FileMover`: 移动文件到目标目录（带重名检测）
5. `EmptyDirCleaner`: 清理空目录（在所有文件处理完成后执行）

核心工具函数：

- `resolve_unique_path(target_path: Path) -> Path`: 处理路径冲突，自动追加 `-Dup1234` 后缀（4位随机数），重试直到无冲突
- `extract_serial_id(filename: str) -> str | None`: 正则提取番号，统一转大写
- `is_video_or_image(path: Path, video_exts: set[str], image_exts: set[str]) -> Literal["video", "image", "other"]`: 判断文件类型
- `find_empty_dirs(root: Path) -> list[Path]`: 递归查找空目录（自底向上排序）

**设计亮点**：

- `resolve_unique_path()` 是纯函数，接受目标路径，返回唯一路径
- `FileRenamer` 和 `FileMover` 都调用此函数，实现逻辑复用
- 重试逻辑封装在 `resolve_unique_path()` 内部，对外透明

## 4. 错误处理

- 每个 Processor 内部捕获异常，返回 `ProcessorResult(status='error', message=...)`
- Pipeline 跳过失败项，继续处理下一个文件
- 最终报告中统计所有成功/失败/跳过的文件

## 5. 日志与报告

- **实时日志**: 每处理一个文件，输出一行 JSON Lines 到 `logs/{task_id}.jsonl`
- **Markdown 报告**: 任务完成后生成 `reports/{task_id}.md`，包含统计图表和详细列表

### 8.2 可测试性设计原则

#### 纯函数优先

所有工具函数设计为纯函数（无副作用），便于单元测试：

```python
# ✓ 易测试
def extract_serial_id(filename: str) -> str | None:
    """从文件名提取番号（纯函数）"""
    pattern = r"([A-Za-z]{2,5}-\d+)"
    match = re.search(pattern, filename)
    return match.group(1).upper() if match else None

# 测试用例
def test_extract_serial_id():
    assert extract_serial_id("ABCD-123.mp4") == "ABCD-123"
    assert extract_serial_id("video_ABC-001_hd.mp4") == "ABC-001"
    assert extract_serial_id("no_serial.mp4") is None
```

#### 依赖注入

Processor 不直接依赖文件系统，通过 Context 传递：

```python
class FileClassifier:
    def __init__(self, video_exts: set[str], image_exts: set[str]):
        self.video_exts = video_exts
        self.image_exts = image_exts
    
    def process(self, ctx: ProcessingContext) -> ProcessorResult:
        ext = ctx.file_info.path.suffix.lower()
        if ext in self.video_exts:
            ctx.file_type = "video"
        # ...
```

测试时可以轻松 mock：

```python
def test_file_classifier():
    classifier = FileClassifier({".mp4", ".avi"}, {".jpg", ".png"})
    ctx = ProcessingContext(file_info=FileInfo(path=Path("test.mp4")))
    result = classifier.process(ctx)
    assert ctx.file_type == "video"
```

#### 文件系统隔离

集成测试使用 pytest 的 `tmp_path` fixture：

```python
def test_file_renamer(tmp_path):
    # 创建临时测试文件
    test_file = tmp_path / "old_name.mp4"
    test_file.write_text("test")
    
    # 执行重命名
    renamer = FileRenamer()
    ctx = ProcessingContext(
        file_info=FileInfo(path=test_file),
        target_path=tmp_path / "new_name.mp4"
    )
    result = renamer.process(ctx)
    
    # 验证结果
    assert (tmp_path / "new_name.mp4").exists()
    assert not test_file.exists()
```
